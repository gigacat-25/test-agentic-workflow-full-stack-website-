    Add conversation management logic to WhatsApp provider.

    **Conversation state tracking:**
    - For simplicity, track conversation state in-memory using a Map (string -> { state, data, expiresAt })
    - States: "IDLE", "AWAITING_ISSUE", "AWAITING_DATE", "AWAITING_TIME", "AWAITING_CONFIRMATION", "AWAITING_FEEDBACK"
    - Each conversation has a TTL (30 min), cleaned up periodically

    **Conversation handler function:**
    `async function handleIncomingMessage(from: string, text: string, services: AllServices, provider, clinicName): Promise<string>`
    
    - Normalize text: lowercase, trim
    - Get or create conversation state for this number
    
    - If state is IDLE:
      - Check for keywords: "book", "appointment", "schedule" -> transition to AWAITING_ISSUE, reply: "What is the issue? Skin, hair, or other?"
      - Check for "cancel" -> find patient by phone, find upcoming appointment, cancel it, reply with confirmation
      - Check for "feedback" or numeric rating -> store feedback if there's a recent completed appointment
      - Otherwise: reply with menu: "Welcome to {clinicName}! Reply: BOOK to book an appointment, CANCEL to cancel, or FEEDBACK to leave feedback."
    
    - If state is AWAITING_ISSUE:
      - Match: "skin", "hair", "other" (fuzzy match)
      - Store serviceType in conversation data
      - Transition to AWAITING_DATE
      - Reply: "Thanks! What date would you prefer? (e.g., 'tomorrow', 'Monday', or a specific date like 'June 15')"
    
    - If state is AWAITING_DATE:
      - Parse date from text: handle "today", "tomorrow", day names ("Monday"), or specific dates
      - Use simple date parsing logic
      - Store date in conversation data
      - Transition to AWAITING_TIME
      - Reply: "What time works best? Morning (9-12), Afternoon (12-4), or Evening (4-6)?"

    - If state is AWAITING_TIME:
      - Match time preference: "morning"/"9-12", "afternoon"/"12-4", "evening"/"4-6"
      - Check availability for the selected date/time
      - If available, propose a specific slot
      - Transition to AWAITING_CONFIRMATION
      - Reply: "Great! We have a slot on {date} at {time}. Reply YES to confirm or NO to try a different time."
    
    - If state is AWAITING_CONFIRMATION:
      - If "yes" or "confirm":
        - Find or create patient by phone number
        - Create appointment with confirmed status
        - Send confirmation: "Your appointment is confirmed! Date: {date}, Time: {time}. We'll send you a reminder. Reply CANCEL if you need to change."
        - Also send email confirmation if email on file
        - Clear conversation state
      - If "no" or "different":
        - Transition back to AWAITING_TIME or AWAITING_DATE
        - Reply: "No problem. What time works better for you?"
    
    - If state is AWAITING_FEEDBACK:
      - Store the response as feedback text
      - Check if the text contains a number 1-5 (extract as rating)
      - Reply: "Thank you for your feedback! We appreciate it."
      - Clear conversation state

    **WhatsApp webhook update:**
    - The webhook handler (in routes/webhooks.ts) should:
      - Parse the incoming message
      - Extract from number and text
      - Call handleIncomingMessage
      - Return the response text via the WhatsApp provider
      - Return 200 OK to the webhook sender

    **Helper functions:**

    - `parseTimePreference(text: string): { start: string; end: string } | null`
    - `formatDateTime(date, start, end): string` — for human-readable messages
