    **index.html:**
    - Standard HTML5 with div#root
    - Title: "SkinCare Clinic — Staff Dashboard"
    - Google Fonts (Inter)
    - Meta viewport

    **src/main.tsx:**
    - React 18 createRoot
    - Import dashboard.css
    - Wrap App in AuthProvider

    **src/auth.tsx:**
    - AuthContext with: { staffUser, token, login, logout, isAuthenticated, isLoading }
    - AuthProvider component:
      - On mount: check localStorage for saved token
      - If token exists, try to decode and set user (basic client-side check)
      - login(email, password): POST to /api/staff/auth/login, store token + user in state and localStorage
      - logout(): clear state + localStorage
    - useAuth() hook
    - ProtectedRoute component: checks isAuthenticated, redirects to /login if not
    - Token storage in localStorage (acceptable for MVP, document upgrade path)

    **src/api.ts:**
    - All API functions accept token as param (or use AuthContext)
    - Factory: createApi(token: string) returns typed API client
    - `login(email, password)`: POST /api/staff/auth/login -> { token, staffUser }
    - `getAppointments(date?, status?)`: GET /api/staff/appointments -> { appointments }
    - `checkIn(id)`: POST /api/staff/appointments/{id}/check-in -> { appointment }
    - `complete(id)`: POST /api/staff/appointments/{id}/complete -> { appointment }
    - `cancelAppointment(id)`: POST /api/staff/appointments/{id}/cancel -> { appointment }
    - `getPatient(id)`: GET /api/staff/patients/{id} -> { patient, appointments, feedback, followUps }
    - `searchPatients(query)`: GET /api/staff/patients?q={query} -> { patients }
    - `getFollowUps(status?, type?)`: GET /api/staff/follow-ups -> { followUps }
    - `createFollowUp(data)`: POST /api/staff/follow-ups -> { followUp }
    - `completeFollowUp(id, notes?)`: POST /api/staff/follow-ups/{id}/complete -> { followUp }
    - `confirmAppointment(id)`: POST /api/staff/appointments/{id}/confirm -> { appointment }
    - All functions handle errors, throw typed errors
    - Use fetch() with Authorization: Bearer {token}

    **src/pages/Login.tsx:**
    - Centered card layout with clinic logo/name
    - Email input + Password input
    - "Sign In" button
    - Error message display for invalid credentials
    - Loading state during login
    - On success: redirect to /dashboard
    - Clean, professional design matching clinic branding

    **src/App.tsx:**
    - React Router (HashRouter for CF Pages compatibility)
    - Routes:
      - /login -> Login (no auth required)
      - / -> ProtectedRoute -> Layout -> Dashboard
      - /patients/:id -> ProtectedRoute -> Layout -> PatientDetail
      - /follow-ups -> ProtectedRoute -> Layout -> FollowUps
    - Default redirect: / -> /login if not auth, / -> /dashboard if auth

    **src/styles/dashboard.css:**
    - CSS custom properties: same palette as public site + dashboard-specific
    - Dashboard layout: sidebar + main content area
    - Card styles, table styles, form styles
    - Status badges (confirmed=blue, checked_in=green, completed=gray, cancelled=red, no_show=orange)
    - Button styles, modal styles
    - Responsive: sidebar collapses on mobile
    - Clean, professional admin aesthetic
