    **src/api.ts:**
    - API base URL: in dev, proxy through Vite or use full URL. For production, same origin.
    - `requestAppointment(data: AppointmentRequest): Promise<AppointmentResponse>`
      - POST to /api/public/appointments/request
      - Handle network errors and API errors gracefully
    - `confirmAppointment(appointmentId: string): Promise<ConfirmResponse>`
      - POST to /api/public/appointments/confirm
    - `cancelAppointment(appointmentId: string): Promise<void>`
      - POST to /api/public/appointments/cancel
    - `getAvailability(date: string, serviceType: string): Promise<TimeSlot[]>`
      - GET /api/public/availability?date=...&serviceType=...
    - All functions handle errors and return typed responses
    - TypeScript types for request/response payloads

    **src/components/AppointmentForm.tsx:**
    - State management with React useState (no form libraries for MVP)
    - Fields:
      - Full name (text input, required)
      - Phone (tel input, required — used for WhatsApp communication)
      - Email (email input, optional)
      - Issue category (select: skin/hair/other, required)
      - Preferred date (date input, required, min=today)
      - Preferred time (select with common slots: Morning 9-12, Afternoon 12-4, Evening 4-6)
      - Additional notes (textarea, optional)
    - Client-side validation:
      - Name: at least 2 chars
      - Phone: basic format validation
      - Date: must be today or future
    - Loading state during submission
    - Error display (inline error messages per field, general error banner)
    - On successful submit: navigate to /confirmation with appointment data
    - On error: show error message, keep form filled
    - Clean form styling matching the main.css design system
    - Responsive layout
