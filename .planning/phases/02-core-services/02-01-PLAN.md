  <action>
    All services receive D1Database in constructor or method params. All IDs are UUIDs generated via crypto.randomUUID(). All use ISO 8601 datetime strings.

    **patient.service.ts** - PatientService class:
    - create(data: { name, phone?, email?, preferred_channel? }): Patient
      - Validate at least one contact method (phone or email)
      - If phone provided, check for existing patient by phone (MATCH prevention)
      - If email provided, check for existing
      - Generate UUID, INSERT into patients table
      - Return created patient
    - findByPhone(phone): Patient | null
    - findByEmail(email): Patient | null
    - findById(id): Patient | null
    - update(id, data): Patient
    - getAppointmentHistory(patientId): Appointment[] (linked from appointment service)

    **availability.service.ts** - AvailabilityService class:
    - isSlotAvailable(date: string, startTime: string, endTime: string, excludeAppointmentId?: string): boolean
      - Query appointments table for overlapping confirmed/checked_in slots on that date
      - Time range: (start_time < endParam AND end_time > startParam)
      - Return true if no conflicts
    - getAvailableSlots(date: string, serviceType: string): TimeSlot[]
      - Define business hours (e.g., 9:00-17:00, 1-hour slots)
      - Check each slot against isSlotAvailable
      - Return array of { start, end } that are free
    - generateTimeSlots(date: string): TimeSlot[] (helper)

    **appointment.service.ts** - AppointmentService class:
    - request(data: { patientId, serviceType, startTime, endTime, source, notes? }): Appointment
      - Validate serviceType is valid
      - Validate source is valid
      - Check availability (throw AppError if slot taken)
      - INSERT appointment with status="requested"
      - Log action in appointment_logs
      - Return created appointment
      - Return any existing appointment that conflicts
    - confirm(appointmentId: string): Appointment
      - UPDATE status="confirmed"
      - Log action
      - Return updated appointment
    - cancel(appointmentId: string, reason?: string): Appointment
      - Find appointment, verify it exists
      - Only cancel if status is "requested", "confirmed", or "checked_in"
      - UPDATE status="cancelled"
      - Log with cancel reason
      - Return updated appointment
    - checkIn(appointmentId: string, staffUserId: string): Appointment
      - Only from status "confirmed"
      - UPDATE status="checked_in", updated_at=now
      - Log action with staff user
    - complete(appointmentId: string, staffUserId: string): Appointment
      - Only from status "checked_in"
      - UPDATE status="completed"
      - Log action
    - noShow(appointmentId: string): Appointment
      - Set status="no_show"
    - getByDate(date: string, statusFilter?: string): Appointment[]
      - Query appointments where start_time LIKE 'date%'
      - Optional status filter
      - JOIN with patients to include patient name
    - getById(id: string): Appointment | null
    - getByPatientId(patientId: string): Appointment[]
    - createManual(data): Appointment (staff creates manually, source="staff_manual")

    **feedback.service.ts** - FeedbackService class:
    - create(data: { appointmentId, rating?, comment?, sentiment? }): Feedback
      - Validate appointment exists and is completed (or just exists for now)
      - INSERT into feedback
      - Return created feedback
    - getByAppointment(appointmentId: string): Feedback | null
    - getByPatient(patientId: string): Feedback[] (via appointment join)

    **follow-up.service.ts** - FollowUpService class:
    - create(data: { patientId, reason, dueDate }): FollowUp
      - Validate patient exists
      - INSERT with status="pending"
    - getByStatus(status: string): FollowUp[] JOIN patients
    - getOverdue(): FollowUp[] (due_date < now AND status='pending')
    - getUpcoming(days?: number): FollowUp[] (due_date between now and +days)
    - complete(id: string, notes?: string): FollowUp
      - UPDATE status="completed", notes
    - getByPatientId(patientId: string): FollowUp[]

    All methods use parameterized queries ($1, $2, ...) for D1 (D1 uses ? or $N binding depending on driver — use ? for positional).
    All throw AppError from utils/errors.ts on validation failure or not-found.
  </action>
